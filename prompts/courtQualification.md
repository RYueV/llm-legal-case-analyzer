Ты выполняешь извлечение структурированных данных из подготовленных материалов судебного акта по делу о банкротстве.

Заполни только блоки:

- qualification
- motivation

согласно переданной JSON Schema.

Работай только с переданными данными.

Входные данные могут содержать:

1. Контекст дела — реквизиты судебного акта, сведения о должнике, краткое описание спора.
2. Карта участников — лица и организации с ролями.
3. Основной материал — фрагменты судебного акта, иногда с source.

Приоритет источников:

Основной материал > Карта участников > Контекст дела.

Общие правила:

- не используй внешние знания;
- не додумывай факты;
- не подменяй вывод суда собственным выводом;
- не превращай правовую позицию суда в установленный факт;
- не превращай установленный факт автоматически в итоговую квалификацию;
- не превращай факт поведения автоматически в мотивировку.

Для неподтвержденных полей:

value = "unknown"
evidence_quote = ""
evidence_source = "none"
confidence = "low"

Для binary-полей используй только:

yes
no
unknown

evidence_quote:
короткая дословная цитата, прямо подтверждающая значение.

evidence_source:
используй только переданный source без изменений.

confidence:

high — есть прямой вывод суда.
medium — требуется осторожная интерпретация.
low — данных недостаточно.

==================================================
ВАЖНЫЕ РАЗГРАНИЧЕНИЯ
==================================================

Qualification описывает итоговый юридический результат.

Motivation описывает логику рассуждений суда.

Не смешивай эти уровни.

Получение преимущества не означает автоматически субординацию.

Предоставление финансирования не означает автоматически компенсационное финансирование.

Аффилированность не означает автоматически участие в управлении.

Обсуждение кризиса не означает автоматически, что кризис стал причиной решения.

Упоминание обстоятельства не означает, что суд признал его доказанным.

Упоминание обстоятельства не означает, что суд сделал его решающим.

Если суд обсуждает несколько возможных оснований исхода, не выбирай одно из них как key_reason_category без прямой опоры.

Поведение кредитора может быть установлено судом, но не иметь значения для итоговой квалификации.

Мотивировка суда может обсуждать обстоятельство, которое в итоге не влияет на результат дела.

==================================================
QUALIFICATION
==================================================

legal_qualification

Используй только итоговый вывод суда.

Не определяй квалификацию самостоятельно.

Возможные значения:

included_general
subordinated
refused
current_claim
corporate_claim
recognized_as_compensation_financing
remand
unknown

Не определяй квалификацию по отдельным признакам или обсуждаемым теориям.

==================================================
СПЕЦИАЛЬНОЕ ПРАВИЛО ДЛЯ REMAND
==================================================

Если кассация отменила акты и направила спор на новое рассмотрение:

legal_qualification обычно = remand

key_reason_category обычно = procedural_remand

Даже если подробно обсуждаются кризис, компенсационное финансирование, аффилированность или рыночность.

==================================================
KEY_REASON_CATEGORY
==================================================

Укажи главную причину исхода.

Используй только причину, которая объясняет итоговое решение суда.

Если несколько причин равнозначны и суд не выделяет главную:

mixed_reasons

Возможные значения:

compensation_financing
corporate_nature
sham_or_artificial_debt
reality_not_proven
no_crisis_at_relevant_moment
market_behavior
abuse_of_control_over_procedure
no_affiliation_or_control_proven
procedural_remand
mixed_reasons
other
unknown

Не выбирай категорию только потому, что соответствующий вопрос обсуждался в мотивировке.

==================================================
ТЕКСТОВЫЕ ПОЛЯ
==================================================

final_decision_text

Краткое описание итогового решения.

Не более 30 слов.

decisive_features_text

1–3 решающих признака.

Не более 30 слов.

Перечисляй через точку с запятой.

non_material_features_text

Обстоятельства, которые обсуждались, но не стали решающими.

Не более 30 слов.

Если таких обстоятельств нет — unknown.

==================================================
MOTIVATION
==================================================

motivation_facts_detailed = yes, если суд подробно анализирует фактические обстоятельства.

motivation_affiliation_only = yes используй крайне редко: только если мотивировка фактически сводится к аффилированности без анализа кризиса, доказательств, экономического смысла, рыночности или поведения.

motivation_economic_sense = yes, если анализируется экономический смысл сделки либо поведения.

motivation_crisis = yes, если реально анализируются кризис, момент кризиса, неплатежеспособность, объективное банкротство либо недостаточность имущества.

Простого упоминания недостаточно.

motivation_independent_creditors = yes, если выводы связываются с защитой независимых кредиторов либо конкурсной массы.

motivation_market_terms = yes, если анализируются проценты, сроки, обеспечение, рыночность условий либо коммерческая логика сделки.

Простого упоминания недостаточно.

motivation_good_faith = yes, если анализируются добросовестность, злоупотребление правом либо обход закона.

motivation_formalistic = yes, если вышестоящий суд критикует формальный подход нижестоящих судов.

motivation_contradictory = yes, если суд прямо указывает на противоречивость выводов.
