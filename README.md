## Назначение

Система преобразует текст судебного акта в структурированное представление, содержащее юридически значимые признаки:

- сведения о деле;
- характеристики требования кредитора;
- признаки аффилированности и контроля;
- сведения о кризисе должника;
- доказательства и оценку суда;
- итоговую юридическую квалификацию.

Дополнительно выполняется контроль качества извлечения и дозаполнение недостающих признаков.

---

## Архитектура

Обработка документа состоит из нескольких этапов:

1. Извлечение текста из PDF.
2. Структурное разбиение акта на зоны.
3. Разбиение текста на предложения.
4. Извлечение служебного контекста:
   - реквизитов дела;
   - правовых ссылок.
5. Формирование временных окон документа.
6. Очистка окон от процессуального шума.
7. Классификация окон в смысловые карточки.
8. Объединение результатов классификации.
9. Извлечение признаков специализированными LLM-экстракторами.
10. Контроль качества извлечения.
11. Дозаполнение недостающих полей (при необходимости).
12. Формирование итоговой структуры дела.

## Основные компоненты

| Модуль                  | Назначение                       |
|-------------------------|----------------------------------|
| pdfParser               | Извлечение текста из PDF         |
| structureSplitter       | Разделение акта на смысловые зоны|
| sentenceExtractor       | Разбиение на предложения         |
| headerExtractor         | Извлечение реквизитов дела       |
| referencesScanner       | Поиск правовых ссылок            |
| dateWindowBuilder       | Формирование текстовых окон      |
| dateWindowSanitizer     | Удаление процессуального шума    |
| windowAnnotation        | Классификация окон               |
| evidenceAwareMerger     | Объединение карточек             |
| schemaExtraction        | Извлечение признаков             |
| extractionAuditor       | Контроль качества                |
| evidenceContextExpander | Подбор дополнительного контекста |
| missingFieldRefiner     | Дозаполнение недостающих полей   |

## Итоговая структура данных

Результатом работы системы является JSON-документ, содержащий:

- metadata;
- procedure;
- claim;
- creditor_profile;
- crisis;
- behavior;
- market_terms;
- proof;
- proof_burden;
- qualification;
- motivation;
- legal_references;
- audit.

Поле `economic_dispute_basket` определяется детерминированно на основе остальных признаков и не заполняется языковой моделью напрямую.
